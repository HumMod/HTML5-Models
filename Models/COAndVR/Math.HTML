The challenge was outlined in the Overview document.

We have cardiac output (CO) as a function of right atrial pressure.

(1) CO = f (RAP) * HS

I'll rearrange Equation (2) to get right-atrial pressure (RAP) on the left-hand side. Now we have RAP as a function of venous return (VR).

(4) RAP = MCFP - VR * RVR

It is likely that cardiac output (CO) will never be exactly equal to venous return (VR) as specified in Equation (3).

(3) CO = VR

So we'll change the rules a bit and say that we have a solution when CO is nearly equal to VR. That is

(5) | VR - CO | <= Error Limit

The absolute value brackets indicate that we don't care about the sign of the error. We only care about the magnitude of the error.

The model uses an implicit algebraic equation solver to do all the work. I arbitrarily set the error limit at 10.0. You'll notice when using the model that the difference between CO and VR is alway 10 or less.

The calculation sequence is as follows.

  • Begin with a trial value of VR.

  • Use Equation (4) to calculate RAP as a function of VR.

  • Use Equation (1) to calculate CO as a function of RAP.

  • Use Equation (5) to compare VR and CO. If the absolute value of the difference is less than the error limit, we have a solution. If the difference is greater than the error limit, pick a better value of VR and repeat these steps.
